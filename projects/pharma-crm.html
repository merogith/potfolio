Pharmaceutical CRM Sales Analytics

Do sales calls actually drive prescriptions — and which doctors deserve the rep's limited time?

View the notebook GitHub folder

The question

A pharma sales team has finite rep time, and two questions decide how to spend it. First: do calls actually move prescriptions, or are reps simply visiting doctors who would have prescribed anyway? Second: which doctors should the team prioritize?

This mirrors my real work at Viatris, where I maintained healthcare-professional records across 70+ countries — so the project is a clean, shareable version of a problem I've handled with live data.

The data

A synthetic but realistic CRM dataset for 600 physicians — call activity and prescription volumes — generated by seeded scripts that deliberately inject real-world messiness (so the cleaning steps are real, not decorative).

Approach & method

  • Cleaned and joined the call and prescription records.
  • Tested the call → prescription relationship with correlation and regression.
  • Segmented the 600 doctors into four behavioral groups with KMeans.
  • Built a ranked priority list using SQL RANK() OVER window functions.

What I found

  • Calls and prescriptions are meaningfully related: r = 0.49 (p = 1.5e-37) — each additional call per year is worth about 5.5 more scripts. The effort isn't wasted.
  • KMeans separates the doctors into four distinct segments with different value and responsiveness.
  • From those, a 99-doctor high-potential list emerges — the accounts where extra calls should pay off most.

The analysis turns a vague "visit more doctors" into a specific, ranked list of 99 to visit first.

Why it matters

It's the full commercial-analytics loop — cleaning, statistical testing, an ML segmentation, and a SQL-built deliverable — all framed around a decision a sales leader actually makes. Exactly the kind of CRM / commercial analytics that data teams run in pharma and beyond.
