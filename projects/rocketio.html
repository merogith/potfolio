Game

RocketIO

A real-time hex-grid strategy game where you command one of 14 world powers.

The engineering: a real-time engine in vanilla JS on HTML5 Canvas — fog of war, a missile economy and adaptive AI — with maps, units and music generated procedurally in code, tested under Vitest and CI.

Plays in your browser, no install

How to play

You pick one of 14 world powers and play in real time on a hex map. Expand your borders, run a missile economy (factories build, launchers fire, anti-air intercepts), and read the map through fog of war, where you only see what your units can see. There are five ways to win, plus a single-player campaign and a tutorial to get started.

Up to six AI opponents share the map with you. They don't all play the same: each runs a different doctrine, and they shift their plans in response to what you're doing, so a rush plays out very differently from a slow build.

Under the hood

The interesting constraint here is that almost nothing is a fixed asset. There are no image files, no pre-built maps, and no recorded music. It's vanilla JS modules and an HTML5 Canvas, and the game assembles its world in code at runtime. Two parts of that are worth pulling apart.

Procedural map generation. Every match starts from a fresh map built with gradient noise: the same technique that gives terrain a natural, rolling shape instead of random static. By tuning it, the generator produces recognisably different worlds (solid continents, scattered archipelagos, inland seas), so the strategy changes with the geography rather than the board being the same every time.

AI that adapts. Each opponent follows a doctrine (an aggressive missile-spammer plays nothing like a turtling defender) and re-weighs its priorities as the game state changes. Because the world is generated, the AI can't memorise an opening; it has to respond to the map and to you, which keeps single-player matches from feeling scripted.

Generating everything in code is harder up front, but it means there's no asset bottleneck: a new map type or unit is a few lines, not a pile of art. Most of the work, then, is in the tuning.

Even the audio is generated through WebAudio as an adaptive score that shifts with the action. Vite handles bundling, Vitest and GitHub Actions run the tests on every push, and a separate balance-simulator tool plays factions against each other so I can tune them from data instead of guesswork.
